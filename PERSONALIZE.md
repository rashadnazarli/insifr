# Personalize insifr-founder-mode

Make this repo yours — replace generic “insifr” wording in docs if you publish under your own brand.

## Naming

- Repo name on GitHub can stay `insifr-founder-mode` or match your org.
- Update `README.md` opening line if you want your product name front and center.

## Methodology doc

- `FOUNDER_MODE.md` may reference example org charts or artifact counts — adjust only if something confuses **your** team.

## Routines

Some `routines/` files mention **Windows paths** or old scaffold copy commands (`robocopy`, `c:\...`). For Cursor-first learning, follow `CURSOR_AGENT_PLAYBOOK.md` and ignore those steps unless you deliberately adopt that scaffold.

## Skills

`skills/` is large. As a new coder:

- Start without opening `skills/`.
- When you need depth: `@skills/<folder>/SKILL.md` (e.g. `review`, `qa`).

## Where artifacts live

Use `docs/projects/<project-name>/` so experiments stay organized.
